## Agent skills

## Git workflow

- Create feature branches from `main`.
- Open a pull request with a clear name and description for every change.
- Never commit unstaged files unless the user has explicitly asked you to do so.

### Issue tracker

Issues and PRDs are tracked in GitHub Issues using the `gh` CLI. See `docs/agents/issue-tracker.md`.

### Triage labels

The repo uses the standard five-role triage vocabulary. See `docs/agents/triage-labels.md`.

### Domain docs

This is a single-context repository with its glossary in `CONTEXT.md` and architectural decisions under `docs/adr/`. See `docs/agents/domain.md`.
